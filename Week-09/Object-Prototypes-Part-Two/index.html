Object & Object Prototypes: Part Two

Object Recap

Object Prototypes

Objects are an integral part to JavaScript. In JavaScript, ALMOST everything is an object. That's very cool if you consider that objects give you properties and methods. It's like having extra functionality on every variable, function, string, number, and array (for example) that you place in your code!

Let's look at what properties a variable with a string has: 

let name = "Shaun McKinnon";
let nameProperties = Object.getOwnPropertyNames(name);
console.log(nameProperties.join(', '));

So we returned a block of numbers and "length". The block of numbers represents the character indexes in our string. The length property is the total character count of our string. If we wanted to access a character using one of the index properties, we can simply do this using one of the numbers. 

console.log(name[1]); // "h"
console.log(name[11]); // "n"
console.log(name["length"]); // 1

Pretty cool eh? But we're not done. Let's look at the properties from our person object:

  1. Create the following object:
    let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"],
  talk: function () {
    console.log("I'm 39");
  },
  walk: function () {
    posX += 5;
  }
}
  2. Output all the properties on the object
    let objProps = Object.getOwnPropertyNames(person);
console.log(objProps.join(', '));
// name, age, interests, talk, walk

Hold the phone:
TALK AND WALK AREN'T PROPERTIES!!! They're methods!
But they are! Properties are simply named values. No different than a variable. The value can be any data type: string, number, array, function, or even another object.
